為什麼字型方案都失敗？
Android Chrome 對 data: URI 字型有時會因 CSP（Content Security Policy） 或瀏覽器安全限制拒絕載入。Netlify 預設會加某些 header，可能阻擋 data URI 字型。

Inline SVG 完全繞過這個問題：路徑數據直接在 HTML 裡，無需任何外部資源或字型載入。

如果非要用文字形式（font-face 方案 + Netlify CSP 修正）
問題根源：Netlify 自動加的 Content-Security-Policy header 可能包含 font-src 'self'，阻擋 data: URI 字型。

在 Netlify 根目錄加一個 netlify.toml：
# netlify.toml [[headers]] for = "/*" [headers.values] Content-Security-Policy = "font-src 'self' data:;"
或在 public/_headers 文件：
/* Content-Security-Policy: font-src 'self' data:;
