Source Han Serif TC · 罕見字 U+2137C

如何用 woff2 字型在網頁顯示「」

完整步驟：取得正確字型 → 子集化 → 部署到 Netlify

⚠ 你上傳的 woff2 不包含這個字
檢測發現上傳的 SourceHanSerifTC-VF.woff2 只有 52 個全形英文字母（Ａ-Ｚ, ａ-ｚ）， 已被子集化，不包含 U+2137C。你需要下載 完整原始版 的 Source Han Serif TC，再自行子集化。
1
下載完整版 Source Han Serif TC

需要 Adobe 官方發布的完整 OTF（包含所有 CJK Extension B 字符）：

# 方法 A：從 Adobe GitHub Releases 下載（推薦）
# https://github.com/adobe-fonts/source-han-serif/releases
# 找 "SuperOTC" 或 "SubsetOTF" → TC → SourceHanSerifTC-Light.otf
# 檔案約 22 MB

# 方法 B：用 curl 下載（如果有完整 URL）
curl -L -o SourceHanSerifTC-Light.otf \
  "https://github.com/adobe-fonts/source-han-serif/\
releases/download/2.003R/\
09_SourceHanSerifTC.zip"
# 解壓後找 OTF/SourceHanSerifTC-Light.otf
注意：必須是 .otf 或未子集化的 .ttf， 不能用已子集化的 woff2（子集化過的字型已刪掉罕見字）。
2
子集化：只保留 U+2137C（用 pyftsubset）

安裝工具並生成一個超小的 woff2，只含「𡍼」這一個字：

# 安裝 Python 工具
pip install fonttools brotli

# 子集化：只保留 U+2137C，輸出 woff2
pyftsubset SourceHanSerifTC-Light.otf \
  --unicodes="U+2137C" \
  --flavor=woff2 \
  --output-file=rarechar.woff2

# 預期輸出：約 3-10 KB（只有一個字形）
ls -lh rarechar.woff2

如果你需要同時顯示多個罕見字，可以加多個 codepoint：

# 多個字符
pyftsubset SourceHanSerifTC-Light.otf \
  --unicodes="U+2137C,U+2B820,U+2B821" \
  --flavor=woff2 \
  --output-file=rarechar.woff2
3
把 woff2 放進專案，在 CSS 用 @font-face 載入

推薦的檔案結構：

your-project/ ├── public/ │ └── fonts/ │ └── rarechar.woff2 ← 子集化後的字型 ├── index.html └── netlify.toml

CSS 寫法（放在 <style> 或獨立 CSS 檔案）：

/* 1. 宣告自訂字型，只對 U+2137C 生效 */
@font-face {
  font-family: 'RareChar';
  src: url('/fonts/rarechar.woff2') format('woff2');
  unicode-range: U+2137C;  /* 關鍵：只對這個字生效 */
  font-display: block;
}

/* 2. 在需要顯示罕見字的元素加入 font-family */
body {
  font-family:
    'RareChar',       /* 先用這個處理 U+2137C */
    'PingFang TC',    /* macOS/iOS 其他中文字 */
    'Microsoft JhengHei', /* Windows */
    'Noto Serif TC',  /* Android/Linux */
    serif;
}

HTML 裡的 <link> preload 寫法（可選，加快載入）： 

<link rel="preload" href="/fonts/rarechar.woff2"
      as="font" type="font/woff2" crossorigin>
4
Netlify 設定：解除 CSP 字型限制

這是之前 Android 失敗的真正原因。在專案根目錄加 netlify.toml：

# netlify.toml
[[headers]]
  for = "/*"
  [headers.values]
    # 允許從自己的域名載入字型（'self'）
    Content-Security-Policy = "default-src 'self'; font-src 'self'; style-src 'self' 'unsafe-inline';"

或用 public/_headers 檔案（適合靜態專案）：

/*
  Content-Security-Policy: default-src 'self'; font-src 'self'; style-src 'self' 'unsafe-inline';
為什麼之前 data: URI 字型會失敗？
Netlify 預設 CSP 中 font-src 不包含 data:， 所以 base64 嵌入的字型被瀏覽器拒絕。用獨立 woff2 檔案（font-src 'self'）就不會有這問題。
5
完整 HTML 範例

把以下代碼直接複製，替換字型路徑即可用：

<!DOCTYPE html>
<html lang="zh-Hant">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- 預載字型（可選，提速） -->
  <link rel="preload" href="/fonts/rarechar.woff2"
        as="font" type="font/woff2" crossorigin>

  <style>
    @font-face {
      font-family: 'RareChar';
      src: url('/fonts/rarechar.woff2') format('woff2');
      unicode-range: U+2137C;
      font-display: block;
    }

    body {
      font-family: 'RareChar', 'PingFang TC',
                   'Microsoft JhengHei', 'Noto Serif TC', serif;
    }
  </style>
</head>
<body>
  <p>這個罕見字「𡍼」現在可以在所有裝置正確顯示。</p>
</body>
</html>
