# user story

整個祈福登記的使用情境是這樣開始的:
寺廟的操作者，需要建構一個可以建立寺廟的信眾基本資料的功能，寺廟操作者可以藉由這個功能來對於來寺廟拜拜或者是來參與法會活動的時候可以記錄下他們的基本資料往後再信眾再一次前來寺廟的時候可以很快速的了解信眾的基本資訊包含聯絡人、祖先、陽上人（超渡祖先或者冤親債主的在世人員也可以稱為遺族）、消災人員、生肖、還有填寫資料的時候他是填自己家 也可以幫娘家填寫 能幫朋友填寫 還有一些屬於其他不在剛剛說的範圍內，消災人員及超度需要提供地址，消災人員需要有一位來當這個地址的戶長，在填完這些資料之後祈福登記的基本資料就已經算是填寫完成了，下雨回這位信眾只要再來參加法會或者一些祈福消災的活動只要提供電話號碼就可以很快速的能夠找到這位信眾的資料並且提供他需要參加法會的一些需求，比如可以告訴他上次超渡的祖先是誰誰誰，消災人員有誰誰誰，為祖先超度的有誰誰誰，這次法會有要參加消災及超渡嗎，能快速的完成參加法會的紀錄，參加法會的活動信眾需要收據時系統要提供收據給信眾，一位信眾可能會有自己家的祈福資料，會有太太妻子娘家的祈福資料，有可能也會幫朋友報名這個祈福資料，也有可能是公司企業的祈福資料。
